# MusicAutoFormatter

음악 라이브러리(FLAC, WAV, MP3)를 고음질 MP3로 자동 변환하고, 태그 및 앨범 아트를 관리하며 파일 구조를 그대로 유지해주는 Python 프로그램입니다.

## 🚀 주요 기능

### 1. 고품질 오디오 및 무결성 변환
- `flac`, `wav`, `mp3` 파일을 찾아 `320kbps` 고품질 MP3로 변환합니다. (FFmpeg 사용)
- **원자적 변환 (Atomic Conversion)**: 변환 중에는 `.tmp` 임시 파일로 생성하며, 변환이 100% 성공한 경우에만 최종 파일로 확정합니다. 프로세스 중단으로 인한 '깨진 파일' 생성을 원천 차단합니다.
- **초고속 무결성 검증 (Duration Check)**: 변환 직후 원본과 결과물의 재생 시간(Duration)을 정밀 비교하여, 데이터 누락이나 인코딩 오류가 발생했는지 자동으로 검증합니다. (허용 오차 0.5초 이내)
- **리소스 최적화**: 앨범 아트(비디오 스트림) 처리를 오디오 인코딩에서 분리하여 변환 속도를 높이고 인코더 오류 가능성을 줄였습니다.

### 2. 강력한 메타데이터 복제 (ID3 Tag)
- **모든 태그 유지**: 제목, 아티스트, 앨범 외에도 가사, 작곡가, 트랙 번호, 디스크 번호 등 모든 메타데이터를 최대한 복제합니다.
- **포맷 간 매핑**: FLAC(Vorbis Comment)에서 MP3(ID3v2.3)로의 매핑이 자동으로 수행됩니다.

### 3. 지능형 앨범 아트 관리
- **용량 최적화**: 내장 앨범 아트가 **2MB를 초과**할 경우, 자동으로 `800x800` 해상도로 리사이징 및 압축(JPEG/PNG)하여 용량을 최적화합니다.
- **태그 정제**: 기존에 비일관적으로 들어있던 여러 장의 앨범 아트를 모두 삭제하고, 최적화된 하나의 Front Cover만 삽입합니다.
- **외부 커버 검색 (Exact & Fuzzy)**: 파일 내부에 이미지가 없는 경우, 해당 폴더 내의 `cover.jpg`, `folder.png` 등은 물론, 파일명에 `art`, `album` 등이 포함된 이미지도 자동으로 찾아 삽입합니다.
- **지원 포맷 확대**: 표준 `JPG`, `PNG` 외에도 `WebP`, `JFIF` 포맷의 이미지를 읽어들여 표준 규격으로 변환 및 삽입합니다.
- **로컬 자원 생성**: 온라인이나 파일에서 찾은 이미지는 음악 파일에 삽입될 뿐만 아니라, 이후 다른 플레이어에서의 관리 편의성을 위해 폴더 내에 `cover.jpg` (또는 `.png`) 파일로 자동 저장됩니다.

### 4. 앨범 단위 메타데이터 일관성 유지
- **다수결 기반 통합 (Majority Voting)**: 한 폴더(앨범) 내의 파일들이 서로 다른 앨범명, 아티스트, 연도 정보를 가지고 있더라도, **가장 빈번하게 등장하는 값**으로 모든 파일을 자동 통일하여 완벽한 앨범 구조를 형성합니다.

### 5. 온라인 앨범 아트 및 메타데이터 보정 (NEW)
- **Last.fm API 연동**: 로컬에 이미지가 없는 경우 Last.fm API를 통해 자동으로 검색하고 삽입합니다.
- **초고화질 이미지 자동 업그레이드**: API가 제공하는 기본 화질(300x300)을 넘어, 서버에 저장된 **원본 고해상도 이미지**를 추적하여 가져옵니다.
- **지능형 이름 복원 (Metadata Restoration)**: 앨범명에 `:`, `/` 등 폴더명에 사용할 수 없는 특수문자가 포함된 경우, 검색 결과를 바탕으로 **공식 명칭을 찾아 태그를 복원**합니다. 예: `My Album_ Title` -> `My Album: Title`
- **최적화된 검색 전략**: 언더바(`_`) 치환, 괄호 정보 제거, 전역 검색(Global Search) 등 다양한 기법을 동원하여 매칭률을 극대화합니다.
- **인메모리 캐싱**: 앨범당 단 한 번의 API 호출만 발생하도록 설계되어 속도가 빠르고 API 한도 걱정이 없습니다.

### 6. 베이스 음악 파일 기반 메타데이터 동기화 (NEW)
- **Base Sync 모드**: 앨범 폴더 내에 `base music example` 폴더가 있고 기준이 되는 음악 파일이 존재할 경우, 자동으로 'Sync 모드'로 동작합니다.
- **메타데이터 일괄 복사**: 기준 파일에서 앨범 아트, 아티스트, 앨범명, 연도, 장르, 설명 등을 추출하여 태그가 없는 대상 파일들에 일괄 적용합니다.
- **지능형 파일명 파싱**: 대상 파일명들에서 공통 접두사(Prefix)를 찾아 제거하고, 남은 부분에서 `{트랙 번호}`와 `{곡 제목}`을 정규식으로 자동 추출하여 태그 및 파일명에 반영합니다.
- **예외 처리**: 언더스코어(`_`), 공백, 마침표 등 다양한 구분자를 유연하게 처리하며, 곡 제목이 없는 경우 트랙 번호만으로 깔끔하게 정리합니다.

### 7. 구조 및 자원 관리
- **커스텀 폴더명 앨범 강제 지정**: `--use-folder-as-album` 옵션을 사용하면 파일 내부의 오리지널 태그 대신, 사용자가 임의로 지정한 **상위 폴더명**을 앨범 태그(`TALB`)로 강제 주입하고, 구조를 깔끔하게 다듬어서(최상위 앨범 폴더 병합) 정리합니다.
  ```text
  [Before] (태그: Original_Album_Name)
  input/
  └── My Best Selection/             <-- (Custom Album Name)
      └── Original_Album_Name/       <-- (불필요한 하위 폴더)
          ├── Disc 1/
          │   └── 01. track.flac
          └── cover.jpg
  
  [After] (태그 강제 변경: My Best Selection)
  output/
  └── My Best Selection/             <-- (직관적인 폴더 구조로 병합)
      ├── Disc 1/
      │   └── 01. track.mp3          
      └── cover.jpg
  ```
- **디렉토리 유지**: 입력 폴더의 전체 구조(`Disc 1/`, `Disc 2/` 등)를 출력 폴더에서도 그대로 재현합니다.
- **비음악 파일 보존**: 가사집(PDF), 텍스트(TXT), 이미지(JPG) 등 모든 비음악 파일을 자동으로 복사하여 라이브러리 무결성을 유지합니다.
- **선택적 M3U 백업**: `--backup-m3u` 옵션을 통해 `.m3u` 파일을 `.m3u.bak`으로 이름을 변경하여 안전하게 백업할 수 있습니다.

### 8. 안전성 및 데이터 보호
- **배치 단위 일괄 삭제 (Atomic Deletion)**: 앨범(또는 배치) 내의 **모든 파일이 성공적으로 변환되었을 때만** 원본 파일을 삭제합니다. 단 하나의 파일이라도 변환에 실패하거나 무결성 검증을 통과하지 못하면, 안전을 위해 해당 앨범의 모든 원본 데이터를 그대로 유지합니다.
- **덮어쓰기 방지**: 출력 경로에 디렉토리가 이미 존재할 경우 작업을 중단하여 실수를 방지합니다.
- **표준 규격 준수**: 모든 결과물은 ID3v2.3 표준을 준수하도록 태그가 정돈되어 차량용 오디오나 구형 플레이어에서도 높은 호환성을 보장합니다.
- **Cross-Platform**: Windows와 Linux 환경에서 모두 동일하게 작동합니다.

### 9. 상세 로깅 시스템
- **자동 로그 생성**: `loguru`를 이용해 실행 시마다 타임스탬프가 포함된 로그 파일이 출력 폴더에 자동 생성됩니다. (`music_auto_formatter_YYYYMMDD_HHMMSS.log`)
- **추적 용이성**: 성공 여부, 파일 삭제 내역, 앨범 아트 최적화 과정을 터미널과 파일 양쪽에서 실시간으로 확인할 수 있습니다.

## 🛠 요구 사항
- **Python 3.x**
- **FFmpeg** (시스템 PATH에 등록되어 있어야 함)
- **uv** (권장 패키지 매니저)

## 🗝 API 설정 (Last.fm)
온라인 앨범 아트 검색 기능을 사용하려면 [Last.fm API](https://www.last.fm/api/account/create)에서 API Key를 발급받아야 합니다.

1. 프로젝트 루트에 `.env` 파일을 생성합니다.
2. 다음과 같이 발급받은 키를 입력합니다:
   ```text
   LASTFM_API_KEY=your_api_key_here
   ```

## 📥 설치 및 실행 방법

`uv`가 설치되어 있다면 의존성 관리 걱정 없이 바로 실행할 수 있습니다.

```bash
# 1. 의존성 동기화 (최초 1회)
uv sync

# 2. 프로그램 실행 (run.py 진입점 사용)
uv run run.py

# (선택 사항) 특정 폴더 지정 및 비트레이트 설정
uv run run.py "./MyFLACs" -o "./MyMP3s" -b 320k --backup-m3u

# 3. 앨범 폴더 일괄 처리 (run_batch.py 사용)
# 여러 앨범 폴더가 들어있는 부모 폴더를 대상으로 실행
uv run run_batch.py "./MyAlbums" -o "./FormattedAlbums"
```

## 🏗 프로젝트 구조
- `run.py`: 단일 앨범/디렉토리용 진입점 (CLI 처리)
- `run_batch.py`: 여러 앨범 폴더 일괄 처리용 진입점
- `src/`: 핵심 도메인별로 구조화된 소스 패키지
  - `core/`: 전체 흐름 관리 (`formatter.py`), 설정(`constants.py`), 로깅(`logger_config.py`)
  - `metadata/`: 태그 추출 및 분석(`metadata_processor.py`), 앨범 통합(`metadata_analyzer.py`), 커버 검색(`cover_finder.py`)
  - `processing/`: 오디오 변환(`audio_utils.py`), 이미지 최적화(`image_utils.py`), 파일 미러링(`file_utils.py`)
  - `library/`: 폴더 구조 정리(`library_manager.py`), 파일 스캔(`scanner_utils.py`)

## ⚙ 상세 옵션
- `input`: 입력 디렉토리 경로 (기본: `.`)
- `-o, --output`: 변환된 파일이 저장될 디렉토리 (기본: `output/`)
- `-b, --bitrate`: 타겟 비트레이트 (기본: `320k`)
- `--keep-source`: 변환 및 복사 성공 시에도 원본 파일을 **삭제하지 않고 유지**합니다.
- `--backup-m3u`: `.m3u` 플레이리스트 파일 복사 시 `.m3u.bak`으로 이름을 변경하여 백업합니다. (기본: `False`)
- `--use-folder-as-album`: 파일의 원본 앨범 태그를 덮어쓰고, 입력된 최상단 폴더명(`custom_album_name`)을 앨범 태그로 강제 지정합니다. 불필요한 하위 폴더 계층도 1단계 축소하여 깔끔하게 정리합니다.

---
**Note**: 이 프로그램은 **기본적으로 작업 성공 후 원본 파일을 삭제**합니다. 원본을 유지하고 싶으시다면 반드시 `--keep-source` 옵션을 사용해 주세요. 모든 작업 결과물은 항상 지정된 전용 폴더에 생성됩니다.
